Custom error messages

Overview

fck includes the option for the user to define custom error messages for different error types. This does not extend to each instance of an error, and only covers the different error types, such as runtime error or divide by zero error. When these errors are returned, the error type will be printed along with the error message for that error and optionally an error message specific to the error raised, not the error instance. For example, an invalid syntax error could be raised with the extra detail that a } was expected, or that a comma was expected. These are both invalid syntax errors, but with different contexts. The message relating to the context cannot be altered, but the message pertaining to the invalid syntax error can be altered.
To alter the messages returned with certain errors, a local file is referenced, the values of whican can be freely altered. This file is a text file where each line has the identifier of an error and then either a single string surrounded by either quotation marks or inverted commas, or a comma seperated list of strings defined in the same way.

              
                <error_identifier> = "<custom_message>"
                <error_identifier> = ["<custom_message>", "<custom_message>"]

Single type

If the value given to an error identifier is a single string, this value wll always be returned. If the message is not ended with either a quotation mark or inverted comma, then this will be inferred and added to the end of the line. In this instance, a warning will be raised to alart the user of the error.

              
                <error_identifier> = "<custom_message>"

Multiple types

If the alue given to an error identifier is a list of strings, these will be selected randomly each time a message is needed. the lists defined can be of any length. If a list is defined with no elements, the default message will be chosen. If one of the elements in the list does not end in either a quitatin mark or inverted comma, this will be inserted before the first comma that comes before a quotation mark or inverted comma to attempt to correct for the error. In this instance, a warning will be raised to alart the user of the error.

              
                <error_identifier> = ["<custom_message>", "<custom_message>"]
