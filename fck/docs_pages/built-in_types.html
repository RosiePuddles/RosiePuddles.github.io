Built-in types

This page details the different built-in types within the fck language.

int type

The int type contains an integer value. Upon initialisation, a value can optionally be assigned. If the value given to be assigned is not an integer, the value is converted into an integer in the following way:

  • float: If the value is a float, it will be rounded to an integer

  • sfloat: If the value is an sfloat, the value of the sfloat will be rounded to an integer

  • bool: If the value is a boolean that is true, this will be converted into a 1. A boolean with a value of false will be converted into a 0

If the value is not covered by the above rules, then an error is raised.

              
                int <variable_identifier>
                int <variable_identifier> :: <value>

float type

The float type contains a float value. Upon initialisation, a value can optionally be assigned. If no value is given, the value will be assigned to a default of 0. The value given can be an int, float, sfloat, or bool. If the value given is none of these, then an error is raised.

              
                float <variable_identifier>
                float <variable_identifier> :: <value>

sfloat type

The sfloat type contains a float value and an absolute error in the given value. Upon initialisation, a value can optionally be assigned, and the number of significant figures can also be optionally assigned. If only a value is given, the minimum number of significant figures in the number will be calculated and this will be set as the number of significant figures. If only the number of significant figures are given, when the value is next assigned, which will be the first given value, is rounded to the previousl given number of significant figures. If both a value and number of significant figures are given, the value is rounded to the given number of significant figures. If neither a value of number of significant figures are given, when the value is next altered, the rules previously stated are followed.
Once the value and number of significant figures has been determined, the absoluter error is calulated and stored along with the value. This absolute error is then used in calulations. The value that can be given follows the same rules as float types. The number of significant figures must be a non-zero integer. If it is not, the number of significant values is rounded to an integer. If this rounded value is 0, an error is raised.

              
                sfloat <variable_identifier>
                sfloat <variable_identifier> :: <value>
                sfloat <variable_identifier> :: , <significant_figures>
                sfloat <variable_identifier> :: <value>, <significant_figures>

bool type

The bool type contains a boolean value. Upon initialisation, a value can optionally be assigned. If the value given is neither true or false, the following rules are applied:

  • int, float, or sfloat: If the value is larger than 0, the value will be converted to true. If the value is less than or equal to 0, the value will be converted to false

  • string: If the string is empty then string will be converted into false, otherwise it will be converted into true

If the value is not covered by the above rules, then an error is raised.

              
                bool <variable_identifier>
                bool <variable_identifier> :: <value>

list type

The list type contains list of values that can be of any type, including custom types. Upon initialisation, a list of values can optionally be assigned. This list can be empty.

              
                bool <variable_identifier>
                bool <variable_identifier> :: <value>

To access a member of a list, the square brackets are used. The values inside of the square brackets can be either single values or expressions that evaluate to a single value. The the given value is not a single value, an error is raised. To access a range of values in the list, a colon(:) is used. The expression before the colon is the index of the first item in the range, and the expression after the colon is the index of the last item in the range. Both of these values are optional, and if not given are assumed to be zero, for the first index, and the largest allowed index, for the last index.

              
                >>> list example_list :: [8, 3, 2, -12.6, 8, 0.46, 4]
                >>> # Single index access
                >>> log(example_list[0])
                8
                >>> # Range access
                >>> log(example_list[3:5])
                [-12.6, 8, 0.46]

If either of the given indexes exceed the length of the list, a warning is raised and the index is set to the largest allowed index.

              
                >>> list example_list :: [8, 3, 2, -12.6, 8, 0.46, 4]
                >>> log(example_list[100])
                **********************************************************************
                Warning:
                Index for list was out of range. Changed value to fit list range: List
                index value was too large for this list
                Traceback (most recent call):
                  File <shell>, line 1, in <program>
                log(example_list[100])
                    ^^^^^^^^^^^^^^^^^
                **********************************************************************
                4
                >>>

When accessing a single item or range of items in a list, the indexes must be integers. If they are not integers, then a warning is taised, and the value is rounded to an integer.

              
                >>> list example_list :: [8, 3, 2, -12.6, 8, 0.46, 4]
                >>> log(example_list[2.9])
                *****************************************************************
                Warning:
                Given index for list was a float. Rounded value: List index was a
                float
                Traceback (most recent call):
                  File <shell>, line 1, in <program>
                log(example_list[2.9])
                                 ^^^^^
                *****************************************************************
                2
                >>>

If the first index given is larger than the last index given, the returned list being reversed, since it goes from the given first index to the given last index. This is an equivalent operation to assigning the list range to a list variable and then reversing the new list.

              
                >>> log(example_list[-1:2])
                [0.46, 8, -12.6, 2]
                >>> log(example_list[2:-1].reverse())
                [0.46, 8, -12.6, 2]
                >>>
